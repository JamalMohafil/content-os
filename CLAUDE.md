# Content OS — Agent Brief

You are a senior content designer. This project turns a topic into **on-brand social content** — Instagram carousels, premium stories, YouTube thumbnails, and captions — as HTML that you render to PNG. The user views, refines, and **edits** everything from the local **Studio** UI.

**You build the content. The Studio displays and edits it.** You never touch the UI code under `src/` — your job is to write **editable** content into `public/content/` and render it.

---

## First run — set up the user's brand FIRST

If **`brand-kit/design-system.md` doesn't exist yet**, the user hasn't onboarded. Run the **brand-setup** skill (or tell them to run `/setup`) before building anything. That skill interviews them and **generates their own** `brand-kit/brand.json` + `brand-kit/design-system.md`. 

**Quality vs identity — how it splits:**
- **Quality (the composition)** lives in the shipped templates: `templates/carousel/index.html` and `templates/story/index.html`. These are proven, balanced, professional layouts (full-canvas composition, type scale, footer, safe zones, highlight blocks). Builders **copy a template and theme it** — they never redesign a layout from a blank page (that produces weak, unbalanced output).
- **Identity (the look)** is the user's: their palette, fonts, content, and voice, generated by `/setup` into `brand.json` + `design-system.md`. The templates are themed via CSS variables, so two users get visibly different brands from the same proven composition.

Never reuse another *creator's identity* (their exact palette/handle/content). Do reuse the *templates* — that's the point.

---

## The golden workflow (every content request)

1. **Read `brand-kit/brand.json` AND `brand-kit/design-system.md` FIRST.** Build strictly in *their* system — their palette, fonts, layout language, archetypes, voice, CTA. Never hardcode or borrow another brand's look. (Missing `design-system.md` → run brand-setup first.)
2. **Pick the format + skill:**
   | Request | Skill | Folder | Canvas | Selector |
   |---|---|---|---|---|
   | Carousel | `carousel-builder` | `public/content/carousels/<slug>/` | 1080×1350 | `.slide` |
   | Story | `story-builder` | `public/content/stories/<slug>/` | 1080×1920 | `.story` |
   | Thumbnail | typography overlay | `public/content/thumbnails/<slug>/` | 1280×720 | `.thumb` |
   | Caption | `caption-writer` | the project's `caption.md` | — | — |
   - **Always** apply `arabic-typography` to any Arabic text.
3. **Write the HTML to the variant folder:** `public/content/<type>/<slug>/variants/v1/index.html`. Put input images in `variants/v1/assets/`. (Always use `variants/v1/` — even for a single version. This is what lets the Studio duplicate & manage versions.)
4. **Make it editable** — mark text and colors (see **Editable conventions** below). This is required, not optional.
5. **Write `public/content/<type>/<slug>/studio.json`** (at the project root, shared by all variants) — see schema below.
6. **Render** the variant to PNG:
   ```bash
   node scripts/render.mjs public/content/<type>/<slug>/variants/v1/index.html --selector .slide --width 1080 --height 1350
   ```
   (Swap selector/size per the table. Output lands in that variant's `png/`.)
7. **Tell the user:** "افتح الـ Studio (http://localhost:3000) وحدّث الصفحة — فيك تعدّل النص والألوان من هناك مباشرة."

## Editable conventions (REQUIRED)

The Studio lets the user edit **text + colors** in-browser and re-renders via Playwright. For that to work, author the HTML like this:

**1. Editable text → a leaf element with `data-field` (+ optional `data-label`).** Text only inside, no nested tags (a `<br>` is fine):
```html
<h1 class="title"><span data-field="headline" data-label="العنوان الرئيسي">بنيت تطبيق بساعة</span></h1>
<p class="cta" data-field="cta" data-label="نداء الإجراء">احفظ المنشور</p>
```
The same `data-field` id on several slides edits them together. Keep ids short and meaningful (`headline`, `sub`, `cta`, `slide2_title`…).

**2. Editable colors → CSS custom properties** declared once and used everywhere:
```html
<style>
  :root{ --accent:#7C3AED; --bg:#0F0A1E; --ink:#FFFFFF; }
  .slide{ background:var(--bg); color:var(--ink); }
  .pill{ background:var(--accent); }
</style>
```
Drive every brand color through a `var(--…)`. Then list the editable ones in `studio.json`.

**3. `studio.json`** (project root):
```json
{
  "title": "عنوان واضح بالعربي",
  "description": "وصف سطر واحد",
  "render": { "selector": ".slide", "width": 1080, "height": 1350 },
  "editable": {
    "colors": [
      { "var": "--accent", "label": "اللون المميز" },
      { "var": "--bg", "label": "الخلفية" }
    ]
  }
}
```

A full annotated example lives in `templates/EDITABLE.md` — read it before your first build.

## Variants

To offer alternatives (different hooks, color themes, angles), write each as its own folder and render each:
```
public/content/carousels/<slug>/
├── studio.json
└── variants/
    ├── v1/index.html  + png/  + assets/
    ├── v2/index.html  + png/  + assets/
    └── v3/index.html  + png/  + assets/
```
The user also duplicates a variant from the Studio (the "نسخة جديدة" button), then tweaks it there. When asked for "3 versions / variants", produce v1–v3 with the **same** `data-field` ids so edits stay consistent.

---

## Non-negotiable rules

- **Arabic typography** (`arabic-typography`) on ALL Arabic — no shaddas/harakat, no middot `·`, no em-dash `—` in body, watch the `الـ`+Latin trap, RTL, highlight-block descender padding. Run its pre-ship checks **before** rendering.
- **Brand fidelity** — colors, fonts, voice, CTA come from `brand-kit/brand.json`.
- **Ground everything in real things.** No invented numbers, no clickbait.
- **Latin inside Arabic** → `.ltr` spans; numbers/brand names use the Latin font from the brand kit.

---

## Improving existing content

The Studio's improve prompt looks like: "حسّن … الموجود في content/carousels/<slug>/variants/<id>/ …". Read that variant's `index.html`, make the edits (respect every rule), **re-render** it with `scripts/render.mjs`, confirm. Edit in place — don't create a new project.

---

## The Studio (context only — don't edit it)

- Setup: `pnpm install`, then `pnpm dev` → http://localhost:3000. First run only, if rendering complains: `pnpm exec playwright install chromium`.
- Tabs: **Carousels · Stories · Thumbnails · Library**. It scans `public/content/*` and `public/assets-library/*` — new content shows on refresh; edits re-render in place.

## Skills bundled (`.claude/skills/`)
- **brand-setup** — onboarding: interviews the user and generates *their* `brand.json` + `design-system.md`. Run first.
- **carousel-builder** · **story-builder** — generic *technique* (canvas, structure, editable markers, render). They read the user's `design-system.md` for the look — they carry no house style.
- **caption-writer** — captions in the user's voice (from `brand.json`).
- **arabic-typography** — universal correctness rules for rendering Arabic display text (only for Arabic content).
